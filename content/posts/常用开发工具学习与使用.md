### Docker
{{< collapse title="Docker " >}}
使用与学习别人开发的工具最好的方式是阅读相应的官方文档或是官方设计规范，docker的官方文档网址：https://docs.docker.com
该网站进行实时update，网页简单清晰，文档阅读量不大，目前官网上搭载的AI能基于官方文档生成答案，大大降低了对docker的学习难度

Docker是什么？
Docker 是一个开源平台，它允许开发者将应用程序及其所需的全部“家当”（代码、运行时环境、系统库、配置文件等）打包成一个标准化的单位，称为容器（Container）。
官网那个图标就很生动形象，轮船上面全是打包成货物的集装箱。
举例来说就是：大学宿舍大家互相学习借鉴一下彼此的作业，你的代码在你的电脑上跑得好好的，但发给舍友却因为“环境不一致”（比如 Python 版本不对、少装了个库）而报错，那么
有了 Docker 后就能把程序和它所需的运行环境一起塞进“集装箱”。只要对方电脑装了 Docker，这个集装箱就能以完全一样的方式运行。



为什么我要使用Dockers？
在这里我先分享一段我一开始不理解，但随着开发经历变深而越发认可的一段话：
There is an interesting misconception about programmers'tooling.One of the best programmers in the world for some reason use very simple tools. 
And a lot of people assume that it's because they are old and simply have not learnt the modern tools.This is not true.
As you get better as a programmer you realize the importance of simplicity in engineering. You start cutting out the"fat" in everything including your own development setup.A lot of programmers start with very featureful heavy development environments and slowly refine them to a notepad over many decades as they realize what's actually important in Software Engineering.
这段话简单来说就是随着程序员水平的不断提升，他们会逐渐意识到简单在工程中的重要性。我个人的理解是，不仅仅是将功能繁杂且笨重的开发工具链与开发环境进行轻量化，也是将功能繁杂且笨重的开发工具链与开发环境进行重构（build my kafka项目中，我对kafka zookeeper 优化重构为 kafka kraft）,甚至在AI发展十分迅猛的今日，当你想掌握以功能强大的AI为驱动的编程新范式（即vibe AI coding），那么你必须对prompt进行精简以避免冗余的闲聊式prompt引导AI生成冗余输出并积累，导致最后整个项目变得极难维护以及大大降低了扩展性
大多数程序员在他们的开发生涯中，在使用别人写好的软件/工具最大的障碍必然是配置环境。但配置环境带来的折磨会极大地消解你对软件、编程本身的兴趣，Docker 的出现简化了环境配置（可能吧）。简单来说 Docker 使用轻量级的“容器”（container）而不是整个操作系统去支持一个应用的配置。应用自身连同它的环境配置被打包为一个个 image 可以自由运行在不同平台的一个个 container 中，这极大地节省了所有人的时间成本。

### github
{{< collapse title="github " >}}
github是什么？
现在是一个非常活跃且资源极为丰富的开源交流社区，但对开发者来说远不止于此，因为从功能上来说，GitHub 是一个在线代码托管平台。
对一个程序员来说，将信息世界的需求映射到现实世界，将抽象的需求落实到具体实现，以及在vibe coding编程新范式开始逐渐普及的当下，程序员都需要扩宽自己的视野与提高自身眼界
对底层的认识甚至是开发经验就显得愈发重要，那么github在程序员的开发生涯的重要性也跟着水涨船高，全世界的软件开发者在 GitHub 上分享各式各样种类繁多的开源项目。大到工业级的深度学习框架 PyTorch, TensorFlow，小到几十行的实用脚本，既有硬核的知识分享，也有保姆级的教程指导，甚至很多技术书籍也在 GitHub上开源，泡在github里能极大提升开发者的功力，老东西爆出来的焚决不练不行。


如何使用github？
首先我们不得不一并谈谈GitHub 和 Git：
GitHub 是一种基于云的平台，可在其中存储、共享并与他人一起编写代码，
Git 是一种可以智能追踪文件中的更改的版本控制系统。 在你和一组人员同时对同一文件进行更改时，Git 的价值就体现出来了。

### git
{{< collapse title="git " >}}
git是什么？
git 是一款分布式的代码版本控制工具。那么版本控制工具（版本控制系统（VCS））是什么？它是用于追踪源代码（或其他文件与文件夹集合）变化的工具，即：1.帮助维护完整的变更历史，并让多人协作更加顺畅。2.通过一系列“快照”来记录某个目录及其内容的变化，每个快照都封装了顶层目录下所有文件与子目录在某一时刻的完整状态。3.保存相关的元数据，例如是谁创建了快照、提交信息是什么等等。

为什么要学习git？
那当然是有用好用爱用。
个人独自开发：能让你查看项目过去的快照、回到过去的任意状态、记录每次更改的原因、在不同的开发分支上并行工作，尝试不同想法，而不会互相干扰，以及完成更多事情。
团队协作开发：清楚展示每个人做了哪些修改，并提供机制解决多人同时修改同一部分代码时产生的冲突

如何学习以及使用git？
说实话git对初学者并不友好，难以上手，界面更是抽象摸不着头脑，命令更是一堆接着一堆；可作为事实上的行业标准，git底层的设计思想却非常优雅，在这里我不对git复杂的界面以及
成堆的命令进行一一介绍，我非常推荐采用自底向上的方式来学习 Git：先从它的数据模型开始，再对命令行界面进行学习。一旦理解了底层数据模型，就能更清楚地知道每条命令究竟在操作什么，从而真正理解 Git 的行为，为此我非常推荐这篇git tutorial:https://missing.csail.mit.edu/2020/version-control/
